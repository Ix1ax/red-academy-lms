Актуальность, новизна, значимость

Учебные материалы разнесены
по сервисам, а контроль обучения
ведётся вручную.

Актуальность

Фрагментация инструментов обучения

Курсы хранятся в одних системах, проверка заданий — в других, прогресс — в таблицах. Контроль ручной, разграничение прав слабое.

Новизна

Корпоративное обучение + открытые интенсивы в одной системе

Микросервисная LMS с ролевой моделью, организациями, партнёрскими заявками и публикацией курсов в едином контуре.

Значимость

Работающее веб-приложение для колледжа и IT-компании

Централизованное хранение материалов, автоматизация прогресса, выдача сертификатов и снижение ручной нагрузки на кураторов.

Технологическая часть · стек

Каждый слой решает свою задачу
и работает независимо от соседних.

Backend

Spring Boot 3.3.5Java 17 · JdbcTemplate

REST API и бизнес-логика. Параметризованные запросы, централизованная обработка ошибок.

Frontend

React 18 · TSVite · Tailwind

Компонентный интерфейс, статическая типизация, framer-motion, lucide-react.

Хранение

PostgreSQL 16+ MinIO S3

Реляционные данные и JSONB. Бинарные объекты — в отдельном хранилище.

События

RabbitMQ+ Redis cache

Асинхронные события между сервисами. Кэш горячих данных и сессий.

Безопасность

JWT + BCryptaccess · refresh

Подписанный токен с ролью и organizationId. Хеширование паролей.

Шлюз

API GatewaySpring Cloud

Единая точка входа /api/*. Маршруты в 4 сервиса, CORS.

Reverse-proxy

Caddy+ Let's Encrypt

Терминирование HTTPS, автоматический выпуск TLS, проксирование frontend.

Инфраструктура

Docker Composeодин compose.yml

Все контейнеры в одном файле. Воспроизводимый запуск и демонстрация.

Конструкторская часть · безопасность данных

Защита прикладного
и инфраструктурного контуров.

BCrypt
Адаптивное хеширование паролей пользователей
JWT
Access-токен с ролью и organizationId, refresh-токен с признаком отзыва
Ролевая модель
Серверные requireUserId / requireRole на каждом защищённом endpoint
SQL
JdbcTemplate с параметризованными запросами — защита от инъекций
Изоляция
Сервисы в Docker-сети, внешний доступ только через Caddy
HTTPS
Шифрование внешнего трафика · автоматический Let's Encrypt
Секреты
Пароли, JWT-ключ, креды RabbitMQ и MinIO — в переменных окружения
Принцип
Клиент не считается доверенной средой
Все критические проверки дублируются на backend: токен, роль, принадлежность к организации, доступ к ресурсу.
Файлы
Метаданные в БД, объекты в MinIO
Бизнес-данные и бинарные объекты разделены. Готовность к антивирусу и политикам доступа.
Аудит
audit_log + события RabbitMQ
Регистрация ключевых действий пользователя для разбора инцидентов и анализа активности.

Ожидаемые результаты

Цель достигнута: работающая платформа,
пройденные сценарии, масштабируемая архитектура.

01

Реализована LMS-платформа целиком

Регистрация и авторизация, каталог курсов, обучение, тесты, интенсивы с этапами, сертификаты, организации, партнёрские заявки.

Ролевая модель и разграничение доступа

Пять ролей: студент, корп. сотрудник, партнёр, наставник, администратор — каждая со своим интерфейсом и серверной проверкой.

Четыре сервиса + Gateway, единый docker-compose.yml, Caddy на внешнем контуре. Воспроизводимый запуск окружения.

Тестовые пользователи и данные. Точки роста: автотесты API, e2e, расширенный аудит, импорт сотрудников.
